# Terminite ja lühendite sõnastik

Käesolev peatükk koondab dokumentatsioonis kasutatavad mõisted ja lühendid. Kui mõistet kasutatakse dokumentatsioonis läbivalt, tuleb selle tähendus esitada siin üheselt.

## Mõisted ja lühendid

| Mõiste või lühend | Tähendus |
|---|---|
| API | Kokkulepitud tehniline liides, mille kaudu väline süsteem suhtleb Postkast 2.0-ga |
| OpenAPI | API tehnilise kirjelduse standardvorming |
| OpenAPI spec | OpenAPI kirjeldusfail või selle tehniline sisu |
| Endpoint | Konkreetne HTTP rada või operatsioon API sees |
| X-tee | Eesti riigi andmevahetuskiht, mille kaudu saab teenuseid turvaliselt vahendada |
| Swagger | Kasutajaliides või tööriist OpenAPI kirjelduse sirvimiseks ja testimiseks |
| Autentimine | Päringu tegija või osapoole tuvastamine |
| Autoriseerimine | Kontroll, kas tuvastatud osapoolel on õigus soovitud toimingu tegemiseks |
| Header | Päringuga või vastusega kaasas edastatav tehniline lisainfo |
| Identifikaator | Väärtus, mille abil tuvastatakse isik, asutus või konkreetne ressurss |
| UUID | Universaalselt kordumatu identifikaator, mida kasutatakse objektide üheseks eristamiseks |
| HTTP | Veebiprotokoll, mille vastusekoodid kirjeldavad päringu töötlemise tulemust |
| JSON | Levinud andmevorming API sisend- ja väljundandmete esitamiseks |
| JWT | Tunnuse vorming, mida kasutatakse valitud lahendustes päringu tegija tuvastamiseks |
| GovSSO | Riiklik autentimislahendus, mida kasutatakse valitud liidestusvoogudes |

## Kasutuspõhimõte

Kui dokumentatsiooni lisatakse uusi lühendeid või erialaseid termineid, tuleb nende tähendus kanda esmalt käesolevasse sõnastikku.
