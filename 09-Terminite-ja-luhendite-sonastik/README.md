# Terminite ja lühendite sõnastik

## Eesmärk

Käesolev peatükk koondab dokumentatsioonis kasutatavad mõisted ja lühendid. Kui mõistet kasutatakse dokumentatsioonis läbivalt, tuleb selle tähendus esitada siin üheselt.

## Mõisted ja lühendid

| Mõiste või lühend | Tähendus |
|---|---|
| Teenusliides | Kokkulepitud tehniline liides, mille kaudu väline süsteem suhtleb Postkast 2.0-ga |
| Rakendusliidese kirjeldus | Tehniline kirjeldus, mis määrab päringute ja vastuste struktuuri ning teenusliidese käitumise |
| OpenAPI | Rakendusliidese kirjelduse standardvorming |
| X-tee | Eesti riigi andmevahetuskiht, mille kaudu saab teenuseid turvaliselt vahendada |
| Autentimine | Päringu tegija või osapoole tuvastamine |
| Autoriseerimine | Kontroll, kas tuvastatud osapoolel on õigus soovitud toimingu tegemiseks |
| Päis | Päringuga või vastusega kaasas edastatav tehniline lisainfo |
| Identifikaator | Väärtus, mille abil tuvastatakse isik, asutus või konkreetne ressurss |
| Universaalselt kordumatu identifikaator | Tehniline tunnus, mida kasutatakse objektide üheseks eristamiseks |
| Hypertext Transfer Protocol | Veebiprotokoll, mille vastusekoodid kirjeldavad päringu töötlemise tulemust |
| JavaScript Object Notation | Levinud andmevorming teenusliideste sisend- ja väljundandmete esitamiseks |
| JavaScript Web Token | Tunnuse vorming, mida kasutatakse valitud lahendustes päringu tegija tuvastamiseks |
| Valitsemissektori ühtne sisselogimine | Riiklik autentimislahendus, mida kasutatakse valitud liidestusvoogudes |

## Kasutuspõhimõte

Kui dokumentatsiooni lisatakse uusi lühendeid või erialaseid termineid, tuleb nende tähendus kanda esmalt käesolevasse sõnastikku.
