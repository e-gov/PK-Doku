# Teenusliidesed ja rakendusliidese kirjeldus

## Üldpõhimõte

Postkast 2.0 teenusliideste tehniline kirjeldus avaldatakse rakendusliidese kirjelduse kujul. See kirjeldus on päringute, vastuste, väljade, andmetüüpide ja võimalike vastusekoodide osas esmane lepinguallikas.

Käesolev peatükk ei dubleeri tehnilist kirjeldust, vaid koondab liidestuja jaoks loogilise ülevaate teenusliideste rühmadest.

## Teenusliideste rühmad

| Rühm | Kirjeldus | Tehnilise kirjelduse viide |
|---|---|---|
| Teavituste teenusliidesed | Teavituste loomine, küsimine, ajakohastamine ja seotud päringud | `[lisada viide]` |
| Kontaktandmete teenusliidesed | Kontaktandmete, eelistuste ja seotud andmete kasutamine | `[lisada viide]` |
| Süsteemiandmete teenusliidesed | Valitud süsteemiandmete ja haldusandmete kasutamine | `[lisada viide]` |
| X-tee kaudu vahendatavad teenusliidesed | Teenusliidesed, mille kasutamine toimub X-tee vahendusel | `[lisada viide]` |

## Rakendusliidese kirjelduse kasutamine

Rakendusliidese tehnilist kirjeldust tuleb kasutada vähemalt järgmistel eesmärkidel:

- päringu ja vastuse väljade kontrollimiseks;
- kohustuslike väljade tuvastamiseks;
- võimalike vastusekoodide kontrollimiseks;
- näidispäringute ja näidisvastuste kasutamiseks arendustöö käigus.

## Oluline piirang

Rakendusliidese tehniline kirjeldus ei asenda keskkondade, ligipääsude, õiguste, päiste ega muudatuste halduse dokumenteerimist. Need teemad on kirjeldatud käesoleva dokumentatsiooni teistes peatükkides.
